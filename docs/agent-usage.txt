aven agent usage
================

Use refs printed by command output. The stable part is the suffix after the
project prefix:

  APP-7KQ9

The project prefix is display context, not identity. If a task moves projects,
the displayed prefix changes and the suffix remains the same.

Refs:

  * Use the printed qualified ref when possible.
  * Bare suffix refs work when unambiguous.
  * Typed suffix refs must be at least 3 characters.
  * Retry with a longer suffix when aven reports error ambiguous-ref.
  * Do not assume an old project prefix still names the task.

Sync:

  * Normal commands work against the local SQLite database and do not require
    network access.
  * Use aven sync --server <url> for explicit sync.
  * If sync.server_url is configured, aven sync can omit --server.
  * When sync.auth_token is configured, clients send it automatically as a
    Bearer token on sync requests.
  * Private or public server binds require sync.auth_token on the server and
    matching sync.auth_token on clients.
  * The optional daemon only performs background sync. Do not assume it is
    required for local task operations.

Markdown input:

  * Use --description-file or --description-stdin for long descriptions.
  * Use note --file or note --stdin for durable task notes.
  * Notes are append-only and are better for handoff context than scratch work.

Conflicts:

  * Inspect conflicts before resolving them.
  * Use conflict list to find conflicted fields.
  * Use conflict show <ref> --field <field> to see stable variant tokens.
  * Use conflict resolve <ref> <field> --use <variant> only after selecting the
    intended value.
  * Use --value, --value-file, or --value-stdin when neither variant is the
    desired final value.
  * Do not bulk resolve conflicts or pick newest, local, or remote by default.
